Main command's GIT

  • git config --global user.name blabla - встановлює ім'я користувача (автор коммітів)
  • git config --global user.email blabal@bla - встановлює адрес ел.пошти користувача (автора коммітів)
  • git init - створює новий локальний репозиторій із заданим іменем
  • git status - перераховує всі нові або змінені файли, які вимагають фіксації
  • git add - індексує вказаний файл для наступного комміта
  • git commit - вказуємо, що поточний зліпок проекту потрібно зберегти
  • git log - історія коммітів поточної гілки
  • git checkout - переключається на обрану гілку і оновлює робочу директорію до її стану
  • git branch - створює нову гілку
  • git push - загружає всі зміни локальної гілки на віддалений репозиторій
  • git pull - загружає дані з віддаленого репозиторію на локальну гілку
  • git merge - вносить зміни вказаної гілки в основну гілку (об'єднує)
  • git clone - робить копію репозиторію

.gitignore файл - з його допомогою можна ігнорувти перерховані файли і директорії

Advanced

Записати такi команди:

  • git revert - відкат комміта шляхом створення нового, протилежного йому комміту, який потрібно відмінити
  • git reset - відміняє всі комміти після заданого залишаючи всі зміни в робочій директорії
  • git rebase - перезаписує історію всіх коммітів та об'єднює в основну гілку
  • git cherry-pick - можна забрати комміти з однієї гілки в іншу
  • git stash - тимчасово зберігає всі незафіксовані зміни відслідковуваних файлів
